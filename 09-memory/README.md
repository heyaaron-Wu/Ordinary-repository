# 09-memory - AI 记忆与事件日志

## 用途

记录系统运行期间的重要事件、错误修复、数据修正、经验教训等。

## 文件命名规范

- `YYYY-MM-DD.md` - 每日记忆日志
- `special-XXX.md` - 特殊事件记录

## 内容类型

1. **错误修复事件** - 系统错误、配置问题、路径错误
2. **数据修正记录** - 计算错误、累计值调整、历史数据修正
3. **对话教训** - 从用户交互中学到的重要信息
4. **临时上下文** - 需要跨会话保持的信息
5. **系统优化** - 性能改进、流程优化记录

## 使用方式

每日创建新的记忆日志，记录：
- 发生的事件（时间、问题、影响）
- 修复步骤（命令、操作）
- 经验教训（避免再次发生）
