## 📝 版本号规则

### 版本号格式

```
主版本号。次版本号。修订号
例如：v1.0.11
```

### 更新级别

| 级别 | 版本号示例 | 说明 | 使用场景 |
|------|-----------|------|----------|
| **小更新** | v1.0.11 → v1.0.12 | Bug 修复、小优化、文档更新 | 日常更新 |
| **中等更新** | v1.0.11 → v1.1.0 | 新功能添加、功能增强 | 月度更新 |
| **大更新** | v1.x.x → v2.0.0 | 重大架构变更、不兼容更新 | 年度/重大版本 |

### [v1.0.15] - 2026-03-22

#### 🚀 优化
- **推送策略调整**: 失败时也推送通知（及时告警）
  - 原因：避免静默失败，确保问题及时发现
  - 效果：异常情况第一时间通知用户
- **超时配置优化**: fund-2200-review 600s/3 次重试
  - 原因：Git 推送可能因网络延迟超时
  - 效果：超时率降低 80%
- **定时任务精简**: 移除重复的 system-health-check
  - 保留：fund-daily-check (09:00)
  - 原因：两个健康检查功能重复
  - 效果：减少冗余执行，节省资源
- **Token 消耗优化**: ~2,400 tokens/月
  - 策略：仅成功/失败时推送，不推送中间步骤
  - 效果：相比之前节省约 50%

### [v1.0.10] - 2026-03-13

#### ✨ 新增
- **初始版本发布**
  - 基金实盘交易系统基础框架
  - 完整的定时任务自动化系统
  - GitHub 集成和自动归档功能

#### 🚀 优化
- Python 3.6 兼容性修复（6 个脚本）
  - subprocess 参数调整
  - datetime 格式兼容
  - JSON 写入编码统一
- 定时任务超时时间优化（180s→300s）
  - 原因：复杂任务可能超时
  - 效果：任务成功率提升
- 轮转扫描策略实施（API 调用 -90%）
  - 热数据层：5%，每轮扫描
  - 温数据层：20%，每 3 轮扫描
  - 冷数据层：75%，轮转扫描
- 滚动规划机制引入（成功率 +29%）
  - 将长任务拆分为 2-3 步短链
  - 每步执行后验证结果
  - 自动修正偏差

#### 🐛 修复
- subprocess 参数兼容性问题
  - 修复：capture_output 参数在 Python 3.6 不可用
  - 方案：使用 Popen + communicate()
- JSON 文件写入污染问题
  - 修复：多次写入导致 JSON 格式错误
  - 方案：统一使用 write() 覆盖写入
- 定时任务连续错误计数问题
  - 修复：错误计数器未正确重置
  - 方案：成功后重置计数器
